# PRISM Calibration Dataset & Evaluation Export V1

## Purpose

Calibration Dataset & Evaluation Export V1 defines how post-match evaluation results are converted into portable, versioned experiment records for later calibration and benchmark comparison.

This layer stores evaluation evidence. It does not tune models, modify rules, alter decision thresholds, or feed actual results back into the prediction runtime.

## Storage philosophy

Git remains the source of truth for code, schemas, specs, benchmark definitions, and small reference fixtures.

Large or continuously growing evaluation datasets are external artifacts. They may be generated by GitHub Actions or local jobs and stored in object storage, databases, workflow artifacts, or other durable data stores without changing the record contract defined here.

## EvaluationRecord

One row represents one completed historical prediction and its observed result.

Required fields:

- dataset_schema_version
- case_id
- match_id
- competition
- kickoff
- home_team
- away_team
- actual_home_goals
- actual_away_goals
- actual_outcome
- home_probability
- draw_probability
- away_probability
- leading_outcome
- leading_probability
- brier_score
- log_loss
- top1_correct
- scoreline_top3_hit
- decision_action
- selected_market
- candidate_correct
- overall_confidence
- evidence_gate
- prism_version
- runtime_version
- git_commit
- data_version
- rule_version
- model_version
- prompt_version
- session_id

## DatasetManifest

Every export batch includes a manifest containing:

- dataset_schema_version
- export_version
- generated_at
- record_count
- content_sha256
- format
- prism_versions represented
- git_commits represented

The SHA-256 is computed from the exact exported dataset payload bytes, using UTF-8 encoding.

## Formats

### JSONL

- one canonical JSON object per line
- UTF-8
- keys sorted
- compact separators
- newline terminated
- deterministic ordering follows the EvaluationSummary result order

### CSV

- UTF-8
- header row always present
- fixed V1 column order
- newline terminated
- booleans rendered as `true` / `false`
- missing optional values rendered as empty fields

## Determinism

Given the same `EvaluationSummary` and the same explicit `generated_at`, export bytes and manifest hash must be identical.

## Governance constraints

1. Export functions are read-only projections of EvaluationResult.
2. Export functions must not rerun PRISM.
3. Export functions must not alter analytical values.
4. Actual results remain post-match evaluation data only.
5. Dataset schema changes require a new schema version.
6. V1 does not implement automatic calibration, parameter search, rule mutation, or bankroll simulation.

## V1 API

```python
records_from_summary(summary: EvaluationSummary) -> tuple[EvaluationRecord, ...]
export_evaluation_jsonl(summary: EvaluationSummary) -> DatasetExport
export_evaluation_csv(summary: EvaluationSummary) -> DatasetExport
```

`DatasetExport` contains both the exact dataset payload string and its immutable `DatasetManifest`.
